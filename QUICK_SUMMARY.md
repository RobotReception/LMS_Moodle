# 🎯 ملخص سريع: كيف يعمل النظام؟

## في 5 نقاط بسيطة:

### 1️⃣ **البناء (Build)**
```
Dockerfile → صورة Docker تحتوي على:
- PHP 8.1 + Apache
- كود Moodle 4.4
- جميع المتطلبات والإضافات
```

### 2️⃣ **التشغيل (Start)**
```
docker compose up -d →
- MariaDB يبدأ → قاعدة بيانات
- Redis يبدأ → ذاكرة تخزين مؤقت
- Moodle يبدأ → ينتظر قاعدة البيانات
```

### 3️⃣ **التثبيت (Install)**
```
docker-entrypoint.sh →
1. ينتظر قاعدة البيانات تصبح جاهزة
2. يشغل: php install.php مع جميع الإعدادات
3. يُنشئ 400+ جدول في قاعدة البيانات
4. يُنشئ ملف config.php
5. ينشئ حساب المدير
```

### 4️⃣ **التشغيل الدائم (Runtime)**
```
Apache يستمع على المنفذ 80 (داخلي)
↓
Docker يوجه المنفذ 5500 (خارجي) → 80 (داخلي)
↓
عندما تفتح: http://94.136.185.54:5500
يصل الطلب إلى Apache في الحاوية
```

### 5️⃣ **معالجة الطلبات (Request Handling)**
```
متصفح → Apache → PHP → config.php
                    ↓
         ┌──────────┴──────────┐
         ↓                     ↓
    MariaDB               Redis
   (البيانات)            (الكاش)
         ↓                     ↓
         └──────────┬──────────┘
                    ↓
              بناء صفحة HTML
                    ↓
               للمتصفح
```

## 🎬 مثال عملي: ماذا يحدث عندما تسجل دخول؟

```
1. تكتب username + password في المتصفح
   ↓
2. المتصفح يرسل POST request إلى:
   http://94.136.185.54:5500/login/index.php
   ↓
3. Apache يستقبل الطلب
   ↓
4. PHP يعالج login/index.php
   ↓
5. يتصل بـ MariaDB:
   SELECT * FROM mdl_user 
   WHERE username = 'admin'
   ↓
6. يتحقق من كلمة المرور (hash)
   ↓
7. إذا صحيحة:
   - ينشئ session في Redis
   - يرسل cookie للمتصفح
   ↓
8. يعيد توجيهك لصفحة Dashboard
   ↓
9. أنت الآن مسجل دخول ✅
```

## 🔑 المفاهيم الأساسية:

| المفهوم | الشرح المبسط |
|---------|--------------|
| **Container** | "صندوق" معزول يحتوي على التطبيق وكل ما يحتاجه |
| **Image** | "نموذج" أو "قالب" لإنشاء حاويات |
| **Volume** | ربط مجلد في السيرفر بمجلد في الحاوية |
| **Network** | "جسر" للتواصل بين الحاويات |
| **Port Mapping** | توجيه منفذ خارجي إلى منفذ داخلي |
| **Environment Vars** | متغيرات تمرر للحاوية (كلمات مرور، إعدادات) |

## 🎓 للمبتدئين: تشبيهات واقعية

### Container = شقة مفروشة
- كل شقة (container) معزولة
- لها أثاثها الخاص (PHP, Apache)
- يمكن نقلها بكل محتوياتها
- إذا أُتلفت، تبني واحدة جديدة من نفس المخطط (image)

### Volume = مستودع مشترك
- البيانات محفوظة خارج الشقة
- حتى لو غيرت الشقة، المستودع باقٍ
- يمكن أخذ نسخة من المستودع (backup)

### Network = ممر بين الشقق
- MariaDB في شقة
- Moodle في شقة أخرى
- الممر (network) يسمح لهم بالتواصل
